Generate a Go program that I can run as the command line utility `go run ./cmd/mapgen`.

It must accept the following command line flags:

- `--input-land land.png`  (optional, input PNG with a transparent background)
- `--input-outline outline.png`  (optional, input PNG with a transparent background)
- `--output-png hex.png`  (output PNG file name)
- `--output-json json.png`  (optional, output JSON file name)
- `--origin-x 2095`  (X pixel that is the origin of the hex grid that is being created, defaults to input PNG width / 2)
- `--origin-y 615`  (Y pixel that is the origin of the hex grid that is being created, defaults to input PNG height / 2)
- `--render-land`  (boolean, defaults to false, when true, the solid land will be rendered in light grey)
- `--render-outline`  (boolean, defaults to true, when true, the outline will be rendered in magenta)
- `--render-hex-grid`  (boolean, defaults to true, when true, the hex grid will be rendered)
- `--apothem 21.7` (pixel size of the apothem for the hexes the program generates)

At least one of input-land or input-outline must be specified.

* If input-land is not specified, force render-land to false
* If input-outline is not specified, force render-outline to false

The (X,Y) origin is in pixels in the image. This maps to the axial coordinates (0,0) in the hex map. The program will generate the entire hex map, using the given apothem.

The program shall:

- Load the inputs.
- If both inputs are specified, they must have exactly the same dimensions.
- If input-land is specified, create a layer representing the land as a mask
- If input-outline is specified, create a layer represent the outline as a mask
- Generate a hex representation of the map (using Red Blob terms, "flat-top" orientation with odd-columns shifted down), each hex containing a flag that's true when the hex is "inside" the land mmask (meaning the center point of the hex is inside the land mask).
- If output-png is specified, create the output png from the layers, in this order
  - Background (transparent, represents water)
  - Land (light grey, represents the land)
  - Silhouette (magenta, represents the outline of the land)
  - Hex grid (solid black)
- If output-json is specified, create a JSON file with these fields:
  - q (axial Q)
  - r (axial Q)
  - col (column coordinate, (0,0) is top left)
  - row (row coordinate, (0,0) is top left)
  - center_x (pixel, center of hex)
  - center_y (pixel, center of hex)
  - is_land (bool, true if center of hex is in land mask)
  - clipped (bool, true if any part of the rendered hex is off the canvas)

PNG doesn't support layers, so consider these to be logical layers that are merged into the final output image. The command line flags will control the output of layers 2, 3, and 4.
